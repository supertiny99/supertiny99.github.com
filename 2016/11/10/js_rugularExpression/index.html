正则表达式

发表于 2016-11-10 | 分类于 javascript |

正则表达式是一个描述字符模式（字符串规则）的对象。 Javascript 内置 RegExp 对象。 String 和 RegExp 都定义了方法可以用正则表达式进行强大的 模式匹配 和 文本检索与替换 功能。

直接量字符

正则表达式中的所有字母和数字都是按照字面含义进行匹配的。 JavasScript 正则表达式语法也支持非字母的字符匹配。 这些字符需要通过反斜线 () 作为前缀进行转义。

字符 匹配
字母和数字字符 本身
\o NUL 字符
\t 制表符
\n 换行符
\v 垂直制表符
\f 换页符
\r 回车符
\xnn 由十六进制数nn指定的拉丁字符
\uxxxx 由十六进制数xxxx指定的 Unicode 字符
\cX 控制字符^X

正则表达式中许多标点符号具有特殊含义，如果想在正则表达式中使用这些字符的直接量进行匹配，则必须使用前缀 \ 。
想要在正则表达式中按照直接量匹配反斜线本身，则必须使用反斜线将其转义。

字符类

将直接量字符单独放进括号内就组成了字符类。 一个字符类可以匹配它所包含的任意字符。

  • 可以通过 “ ^ ” 符号来定义否定字符类，匹配所有不包含在括号内的字符。
  • 可以使用连字符 “ - ”来表示字符范围。

例如：
/[abc]/ 和字母 ‘a’, ‘b’, ‘c’中的任意一个都匹配。
/[^abc]/ 匹配 ‘a’, ‘b’, ‘c’之外的所有字符。
/[a-zA-Z0-9]/ 匹配拉丁字母表中的任何字母和数字。

字符 匹配
[…] 方括号内的任意字符
[^…] 不在方括号内的任意字符
. 除换行符和其他Unicode行终止符以外的任意字符
\w 任何ASCII字符组成的单词， 等价于[a-zA-Z0-9]
\W 任何不是ASCII字符组成的单词， 等价于[^a-zA-Z0-9]
\s 任何Unicode空白符
\S 任何非Unicode空白符
\d 任何ASCII数字，等价于[0-9]
\D 除了ASCII数字之外的任何字符，等价于[^0-9]
[\b] 退格直接量

重复

在正则模式之后跟随用以指定字符重复的标记。

字符 含义
{n, m} 匹配前一项至少 n 次， 但不能超过 m 次
{n, } 匹配前一项 n 次 或者更多次
{n} 匹配前一项 n 次
? 匹配前一项 0 次或者 1 次， 等价于 {0, 1}
+ 匹配前一项 1 次或者多次， 等价于 {1, }
* 匹配前一项 0 次或者多次， 等价于 {0, }

非贪婪的重复

在匹配的字符后跟随一个问号： ??、 +? 、 *? 或 {1, 5}?。
尽可能少地匹配。

锚点

匹配一个位置

  • ^ : 起始位置
  • $ : 结尾位置
  • \b : 单词边界

选择、分组和引用

正则表达式的语法还包括指定选择项、自表达式分组和引用前一子表达式的特殊字符。

| 用于分割供选择的字符。

例如： /ab|cd|ef/ 可以匹配 ab, cd ,ef 。
选择项的尝试匹配次序是从左到右知道发现了匹配项。

正则表达式的圆括号多种作用。

  1. 把单独的项组合成子表达式。
    例子： /java(script)?/ 可以匹配字符串 “java”，其后面可以有“script”也可以没有。
  2. 在完成的模式中定义子模式。
    当正则表达式成功地和目标字符串相匹配的时候，可以从目标串中抽出和圆括号中的子模式想匹配的部分。（确定捕获部分）。

  3. 允许在同一正则表达式的后部引用前面的子表达式。
    用 \1, \2, \3 来表示带括号的子表达式在正则表达式中的位置。

修饰符

用来说明高级匹配模式的规则。放在 / 符号之外。
JavaScript 支持三个修饰符。

  • i 说明模式匹配是不区分大小写的
  • g 执行一个全局匹配，找到所有的匹配，而不是在找到第一个之后就停止
  • m 多行匹配模式， ^匹配一行的开头和字符串的开头， $ 匹配行的结束和字符串的结束

用于模式匹配的 String 方法

String 支持 4 种使用正则表达式的方法，分别是 search(), replace(), match(), split()。

search()

概要： string.search(regexp)
参数：
regexp 一个 RegExp 对象。 若不是一个 RegExp， 先传入 RegExp() 构造函数， 转换为一个 RegExp 对象。
返回： string中第一个匹配 regexp 的子串的开始位置， 如果没有找到则返回 -1。
描述： search()不执行全局匹配，会忽略g修饰符，忽略regexp的lastIndex属性，总是从string的开始位置开始搜索。

replace()

概要： string.replace(regexp, replacement)
参数：
regexp 指定要替换的模式的 RegExp 对象， 如果是一个字符串， 用作一个要搜索的直接量文本模式；不会先转化为 RegExp 对象。
replacement 一个内容为替换文本的字符串，或者一个函数。
返回： 一个新的字符串， 其中匹配的第一个或者说有地方已替换为 replacement。

match()

概要： string.match(regexp)
参数：
regexp 一个 RegExp 对象。 若不是一个 RegExp， 先传入 RegExp() 构造函数， 转换为一个 RegExp 对象。
返回: 一个包含匹配结果的数组。数组内容取决于 regexp 是否设置了 “g” 属性。
描述：

  • 如果 regexp 没有 “g” 标识， 执行一次匹配， 若无结果，返回 null， 若有结果返回一个数组 [匹配文本,$1,$2,……], 并且有两个对象属性， obj.index 表明位置, obj.input 表明本身string引用
  • 如果 regexp 有 “g” 标识， 执行一次全局搜索， 若无结果， 返回 null， 若有匹配结果返回一个数组， [匹配文本1, 匹配文本2,…]，没有 index 和 input 属性。

split()

概要: string.split(delimiter, limit)
参数：
delimiter string切分处的字符串或正则表达式。
limit 可选整数，指定返回数组的最大长度。
返回： 一个由字符串组成的数组。
String.split()方法是Array.join()方法的反方法。

exec() 方法

概要： regexp.exec(string)
参数: string 要搜索的字符串
返回： 一个包含匹配结果的数组，如果没有找到则为null。
描述：

  • 在一个非全局的RegExp对象上调用时，本函数返回的数组和String.match()方法返回的数组一样。

  • 若RegExp 是一个全局正则表达式。
    从 regexp.lastIndex 指定的位置开始搜索，当找到第一个匹配的项时，将 lastIndex 设置为该匹配之后的第一个字符位置。
    这表明重复调用 exec() 可以循环遍历一个字符串中所有的匹配项。
    若 exec() 找不到匹配项， 它将返回 null 并将 lastIndex 重置为 0 。
    注意： 如果在成功找到一个字符串的匹配项之后，理科开始搜索另一个新的字符串，就必须小心地手动将lastIndex重置为0
